# ソフトウェア設計と実装のルール

- このソフトウェアは、保守性を最大限高めるために、適切な疎結合を意識した実装を行う。
  - オニオンアーキテクチャを意識したレイヤの依存関係を保った実装を行うこと。
  - DDDを意識して、コンテキストごとに適切な疎結合が保たれていること。
  − CQRSを意識して、コマンドとクエリの責務を適切に分割した実装がされていること。
- 実装を行う時は、docフォルダ以下のドキュメントと矛盾のない実装を行うこと。
- このソフトウェアは、モジュラーモノリスアーキテクチャで実装するただし、将来的には、マイクロサービスアーキテクチャでコンテキストを分割することも想定して実装を行うこと。
- テーブル名は将来マイクロサービスで分割すること見越して「コンテキスト名_テーブル名」とすること、ただしこのテーブル名にマッピングするJavaのクラス名にはコンテキスト名は不要とする。

# JavaDocのルール

- 実装を行う時は、今後別の開発者がメンテナンスを行うことを想定したJavaDocを記載すること。
- JavaDocは、日本人の開発者が分かるように日本語で記載すること。
- JavaDocの先頭１行で概要を書きピリオドで終わること。
- CheckStyleの関係で、句点の代わりにピリオドを使うこと。
