# JUnitのテストルール

JUnitのテストを書く際には以下のルールを遵守すること
- テストメソッド名はテストの概要を簡潔に表す英語名前にすること。ローマ字命名は禁止
- 必ずテストメソッドに @DisplayName アノテーションを配置して、そのテストが簡潔に何をしているテストなのかわかる日本の説明を記載すること
- テストメソッド内は、AAA（Arrange-Act-Assert）を意識してテストを記載して、必ず「準備(Arrange)」、「実行(Act)」、「検証(Assert)」のコメントを含めること
- 検証はなるべくassertjのライブラリを使って検証を行うこと。
- 単体テストは、古典学派の考えに則りテストを作成すること。古典学派とは「全てのテストケースはお互いに影響を与えることなく個別に実行できるようにしなくてはならない」という考え方で、DBやファイルシステムなどの共有依存の部分のみをモックにしてテストを実装してください。
