# ECサイト機能一覧

ECサイトのバックエンド機能を、モジュラーモノリス構成の各コンテキストごとに定義します。

## 1. 商品コンテキスト (Product Context)
商品の情報を管理し、カタログとして提供するコンテキストです。

- **商品検索**: キーワード、カテゴリ、価格帯などで商品を検索する。
- **商品詳細**: 商品の詳細情報（名称、説明、価格、画像など）を提供する。
- **カテゴリ管理**: 商品のカテゴリを管理する。

## 2. 注文コンテキスト (Order Context)
顧客の購買活動（カート、注文）を管理するコンテキストです。

- **カート管理**:
    - 商品をカートに追加する。
    - カート内の商品数量を変更する。
    - カートから商品を削除する。
    - カートの内容を確認する。
- **注文**:
    - カートの内容で注文を確定する。
    - 配送先情報を指定する。
    - 決済方法を選択する（決済ゲートウェイ連携は別途検討）。
- **注文履歴**:
    - 過去の注文履歴を確認する。
    - 注文の詳細（配送状況など）を確認する。

## 3. 在庫コンテキスト (Inventory Context)
商品の在庫数を管理するコンテキストです。

- **在庫確認**: 商品の現在の在庫数を確認する。
- **在庫引き当て**: 注文時に在庫を確保（引き当て）する。
- **在庫数更新**: 入荷や出荷に伴い在庫数を増減させる。

## 4. 会員コンテキスト (User Context)
顧客情報と認証を管理するコンテキストです。

- **会員登録**: 新規会員登録を行う。
- **ログイン/認証**: メールアドレスとパスワードでログインし、アクセストークンを発行する。
- **会員情報管理**: 自身のプロフィール（氏名、住所、連絡先など）を参照・更新する。
- **会員削除**: 会員情報を削除（物理削除）する。削除後の復元は不可。
